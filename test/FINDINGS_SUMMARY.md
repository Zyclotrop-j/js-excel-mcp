## Recurring test-harness fixes needed (all in `test/`, no src touch)
1. `testContext = await createTestContext(...)` — several files omit `await`, so `teardown`'s `.cleanup()` fails.
2. Register `CellWriteHandler` (`../../src/tools/handleCells/write.js`) in `setup` whenever `set_cell`/`set_cells` are seeded.
3. Error/no-workbook tests must reuse the SAME `mockServer`+handlers (or wire `.server`+`.context` on the separate handler) — don't build a bare `MockMcpServer`.
4. Use non-overlapping ranges / fresh cells to avoid shared-state leakage between tests.
5. `set-context.test.ts` must follow the `baretest` import pattern used by every other suite.

## Suspected `src` bugs (for follow-up, NOT fixed here)
- **Request-context binding at `register()` time** in `handleChart.ts`, `handlePrint.ts`, `handleRichText.ts` (and likely `handleHyperlink.ts`): handlers close over a single user's `Context`, so the per-request `ctx` userId is ignored and user isolation / no-workbook guards don't work as the tests expect.
- **`handleProtection.ts` `lock_cell`** returns `ref: undefined` (`getCoordinate(cell)` issue).
- **`handleNumberFormat.ts`** cannot locate cells written by `set_cell` (`getCellByCoord` returns undefined → "cell is empty").
- **`handleImage.ts`** has no fetch request timeout (hangs when offline).
